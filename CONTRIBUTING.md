# Contributing

Thanks for improving syncpick. Keep changes small, portable, and safe for
users' local files.

1. Open an issue first for substantial behavior changes.
2. Run `zsh -n syncpick install` and `./tests/test.sh` before opening a PR.
3. Update the README and help text for user-visible changes.
4. Never include real hosts, usernames, IP addresses, paths, or transfer logs
   that reveal private information.

By contributing, you agree that your contributions are licensed under the MIT
License.
